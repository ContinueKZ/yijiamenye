# DESIGN.md

# 益嘉门业设计系统

## 1. 产品定位

益嘉门业官网是一个简约、高级、以产品图片为核心的门业展示网站。

网站不做商城，不支持购物车、在线下单或在线支付。核心目标是让客户通过高质量产品图片、清晰分类、产品描述和联系方式产生咨询意向。

整体体验应该像一个干净、专业、有质感的门业展厅，而不是普通建材批发网站。

关键词：

- 简约
- 高级
- 干净
- 稳重
- 专业
- 图片优先
- 易于联系

## 2. 视觉方向

整体采用现代极简风格，页面留白充足，信息层级清晰，避免复杂装饰。

页面第一视觉应该让用户看到门的质感、空间适配效果和品牌专业度。

推荐气质：

- 精品展厅
- 现代家装
- 高端定制
- 安静可靠
- 清爽克制

避免：

- 低价促销风格
- 花哨渐变背景
- 大量营销标签
- 拥挤的信息排版
- 图片很小、文字很多的产品列表
- 电商购物车、立即购买、在线支付等商城体验

## 3. 色彩系统

### CSS Variables

```css
:root {
  --color-primary: #1f1f1f;
  --color-primary-soft: #3a3a3a;
  --color-background: #f7f5f0;
  --color-surface: #ffffff;
  --color-border: #e4ded4;
  --color-muted: #7a746b;
  --color-accent: #a78352;
  --color-accent-dark: #7c5f38;
}
```

### 使用规则

- `--color-primary` 用于主标题、正文重点、导航和深色按钮。
- `--color-background` 用于整体页面背景，营造温润、干净的高级感。
- `--color-surface` 用于产品卡片、内容区域和弹层。
- `--color-border` 用于细边框、分割线和卡片描边。
- `--color-muted` 用于辅助文字、说明文字和弱信息。
- `--color-accent` 只用于少量强调，例如分类选中、按钮 hover、局部装饰线。

整体颜色必须克制，不要让页面变成大面积金色、棕色或单一暖色调。

## 4. 字体系统

优先使用系统字体，保证中文显示稳定、清晰。

```css
body {
  font-family:
    -apple-system,
    BlinkMacSystemFont,
    "PingFang SC",
    "Microsoft YaHei",
    "Helvetica Neue",
    Arial,
    sans-serif;
}
```

### 字号

```css
:root {
  --font-size-xs: 12px;
  --font-size-sm: 14px;
  --font-size-md: 16px;
  --font-size-lg: 20px;
  --font-size-xl: 28px;
  --font-size-2xl: 40px;
  --font-size-3xl: 56px;
}
```

### 排版规则

- 首页主标题可以有视觉冲击，但不能夸张。
- 产品名称必须清晰、稳定、容易扫读。
- 产品描述控制在 2 到 4 行内，避免长段文字压过图片。
- 导航、按钮、分类名称保持短句。
- 不使用负 `letter-spacing`。
- 不使用过细字体，移动端也必须易读。

## 5. 布局系统

### 容器

```css
:root {
  --container-max: 1200px;
  --container-padding-desktop: 40px;
  --container-padding-tablet: 28px;
  --container-padding-mobile: 20px;
}
```

### 间距

```css
:root {
  --space-xs: 6px;
  --space-sm: 12px;
  --space-md: 20px;
  --space-lg: 32px;
  --space-xl: 56px;
  --space-2xl: 88px;
}
```

### 布局原则

- PC 端内容最大宽度控制在 `1200px` 左右。
- 首页模块之间保持足够留白。
- 产品图片区域优先占据视觉中心。
- 手机端优先单列布局，图片在前，文字在后。
- 平板端使用双列或三列网格，避免内容过宽。
- 固定比例元素要使用 `aspect-ratio`，避免图片加载时页面跳动。

## 6. 三端响应式规则

网站必须兼容：

- PC
- 平板电脑
- 手机

### Breakpoints

```css
:root {
  --breakpoint-mobile: 640px;
  --breakpoint-tablet: 1024px;
  --breakpoint-desktop: 1200px;
}
```

### PC

- 顶部导航横向展示。
- 首页首屏使用大幅产品或空间图片。
- 产品列表建议 3 列或 4 列。
- 产品详情页使用左侧图片、右侧信息的布局。
- 联系入口在首页、产品详情和页脚中保持明显。

### 平板

- 产品列表建议 2 列或 3 列。
- 首页主视觉高度适当降低。
- 图文布局可以保持双列，但要避免文字过窄。
- 导航根据宽度决定是否切换为菜单按钮。

### 手机

- 顶部导航切换为折叠菜单。
- 产品列表优先单列，必要时可以双列，但图片不能过小。
- 产品详情页图片在上，描述和联系方式在下。
- 联系按钮点击区域不小于 `44px`。
- 页面不能出现横向滚动、文字重叠或按钮文字溢出。

## 7. 图片规范

产品图片是整个网站最重要的内容。

### 产品图片规则

- 产品卡片图片比例统一，推荐 `4 / 5` 或 `1 / 1`。
- 产品详情页首图必须足够大、足够清晰。
- 图片不要被过度裁切，门体必须完整或主体明确。
- 图片背景尽量干净，突出门本身。
- 避免模糊、低清、过暗、过度滤镜的图片。
- 不使用卡通图或廉价素材图作为产品图。

### 图片交互

产品详情页需要支持：

- 主图展示
- 多张缩略图切换
- 图片 hover 轻微放大
- 移动端滑动查看多图

### 占位图片

真实产品图未提供前，占位图应保持统一风格：

- 现代室内空间
- 门体清晰可见
- 色调干净自然
- 不使用插画、卡通或抽象图形

## 8. 核心组件规范

### Header

Header 保持简洁、稳重。

包含：

- 左侧品牌名：益嘉门业
- 右侧导航：首页、产品分类、关于我们、联系我们
- 手机端菜单按钮

样式：

- 白色或浅色背景
- 底部细边框
- 滚动时保持清晰
- 不使用厚重阴影

### Hero

首页首屏必须突出品牌和门的视觉。

包含：

- 大幅门类或空间图片
- 品牌标题
- 简短 slogan
- 查看产品按钮
- 联系我们按钮

Hero 不能做成纯文字区域，图片必须是第一视觉信号。

### Product Card

产品卡片用于展示门的图片和基本信息。

包含：

- 产品图片
- 产品名称
- 所属分类
- 简短描述
- 查看详情入口

样式：

- 圆角不超过 `8px`
- 使用轻边框
- hover 时图片轻微放大
- 不使用厚重阴影
- 图片比例稳定，不因文字长度改变卡片高度结构

### Category Filter

产品分类筛选用于快速浏览不同门类。

首版分类建议：

- 全部
- 房门 / 卧室门
- 卫生间门
- 厨房门
- 推拉门
- 无轨推拉门
- 入户门 / 安全门
- 阳台门 / 隔断门

选中状态使用 `--color-accent` 或深色文字加细边框表达。

### Contact Block

联系模块必须在首页、产品详情页、联系我们页出现。

包含：

- 电话
- 微信
- 地址
- 营业时间
- 咨询按钮

联系信息必须来自统一常量，不能散落在多个组件里。

## 9. 按钮规范

### Primary Button

用于主要转化：

- 查看产品
- 联系我们
- 立即咨询

```css
.button-primary {
  background: #1f1f1f;
  color: #ffffff;
  border: 1px solid #1f1f1f;
  border-radius: 4px;
}
```

### Secondary Button

用于次级操作：

- 查看分类
- 了解更多

```css
.button-secondary {
  background: transparent;
  color: #1f1f1f;
  border: 1px solid #cfc7bb;
  border-radius: 4px;
}
```

按钮文字必须短，不写长句。移动端按钮高度不小于 `44px`。

## 10. 动效规范

动画要轻，不能喧宾夺主。

允许：

- 图片 hover 轻微 `scale`
- 按钮 hover 颜色变化
- 页面进入时轻微 `fade up`
- 菜单展开收起

避免：

- 大量滚动动画
- 强烈弹跳动画
- 复杂 3D 转场
- 自动轮播过快

推荐参数：

```css
:root {
  --transition-fast: 160ms ease;
  --transition-base: 220ms ease;
  --transition-slow: 360ms ease;
}
```

## 11. 页面要求

### 首页

首页需要包含：

- Hero 主视觉
- 产品分类入口
- 精选产品
- 品牌优势
- 关于益嘉门业简短介绍
- 联系咨询入口

首页目标是快速建立信任，并引导用户查看产品。

### 产品分类页

产品分类页需要包含：

- 分类筛选
- 产品网格
- 产品图片
- 产品名称
- 产品描述
- 联系入口

产品分类页目标是让用户高效浏览图片，并快速找到感兴趣的门类。

### 产品详情页

产品详情页需要包含：

- 产品主图
- 多图缩略图
- 产品名称
- 所属分类
- 产品描述
- 适用空间
- 材质或风格标签
- 联系方式入口

产品详情页不显示在线购买、加入购物车、立即支付等功能。

### 关于我们

关于我们页面需要表达：

- 益嘉门业是谁
- 主要提供什么类型的门
- 服务哪些客户和空间
- 对品质、安装、定制和售后的重视

文案保持真实、克制，不夸大。

### 联系我们

联系我们页面需要包含：

- 电话
- 微信
- 地址
- 营业时间
- 咨询说明
- 地图占位区域

联系方式变化时，只修改统一常量文件。

## 13. 可访问性要求

基础要求：

- 图片必须有 `alt`。
- 按钮文字必须清楚。
- 链接可被键盘访问。
- 文字和背景对比度必须足够。
- 移动端点击区域不小于 `44px`。
- 不只依赖颜色表达状态。

## 14. 禁止事项

项目中不要出现：

- 在线支付
- 加入购物车
- 立即购买
- 价格结算
- 复杂商城下单流程
- 低价促销弹窗
- 过度营销文案
- 大量装饰性渐变
- 产品图片很小的列表布局

## 15. 成功标准

最终网站应满足：

- 第一眼能看出这是益嘉门业的门类展示官网。
- 产品图片是页面最重要的信息。
- 用户可以快速浏览分类和产品。
- 用户可以很容易找到联系方式。
- PC、平板、手机三端布局自然、稳定、无重叠。
- 视觉简约、高级、有家装展厅感。
- 站点信息、联系方式、产品分类和产品数据都能通过常量统一维护。
